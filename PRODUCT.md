# Product

## Register

product

## Users

Stack Overflow for Teams and Stack Enterprise administrators, community managers, enablement leads, and technical operators who need to run reporting utilities without persisting credentials while keeping report datasets locally available in their browser. They use the app during focused operational workflows: preparing reports, checking user health, importing generated exports, and running scoped live API collections.

## Product Purpose

Stack API Utilities provides browser-ready reporting and write-tool workflows for Stack Overflow for Teams and Stack Enterprise. Success means a user can quickly understand the available reports, add session credentials with confidence, run or upload data, inspect dashboards and raw records, and leave with trustworthy outputs.

## Brand Personality

Trusted, practical, sharp. The interface should feel like a Stack Overflow-native operations console: clear, technically confident, respectful of dense data, and warmer than a generic admin dashboard.

## Anti-references

Avoid low-contrast dark surfaces, nearly invisible Stacks buttons, generic SaaS card grids, decorative dashboard noise, and visual choices that obscure the user’s current task. Do not copy the Stack Overflow corporate landing page as a marketing hero; translate its emphasis on trusted knowledge into product UI.

## Design Principles

- Make the next action obvious without turning every surface into a CTA.
- Treat credentials as sensitive session state, treat datasets as browser-local sensitive data, and make both constraints visible.
- Keep dense reporting scannable through consistent panels, tables, and metric treatments.
- Use Stack Overflow orange as a functional accent for primary action, selection, and focus.
- Prefer familiar product affordances over ornamental novelty.

## Accessibility & Inclusion

Target WCAG AA contrast for text, form fields, buttons, and focus states. Maintain visible focus rings, clear disabled states, keyboard-operable navigation, and reduced-motion-safe transitions. Avoid color-only status communication where labels or structure can carry the meaning.
